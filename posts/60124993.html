语音电话技术之CUCM配置

发表于2024-12-15|更新于2025-09-21|AI-X运维语音电话/视频会议

一、CUCM服务开启

进入到Cisco Unified Serviceability下

进入Tools→Service Activation

可以根据需求开启某些服务，一般方便起见可以通过Check All Services然后点击Save先把所有服务都起来

二、注册话机

自动注册

进入到System→ Cisco Unified CM下

找到需要开启自动注册CUCM，并点击进入

在Auto-registration information中填写起始号码和结尾号码，并将Auto-registration Disable on this Cisco Unified Communications Manager的“√”去掉。

此时TFTP地址指向该CUCM的IP话机将自动注册到此CUCM上，号码从1000开始往下下发，当设置的自动注册号码段全部被使用时，自动注册将不再生效，需要重新定义可用的号码段后才会继续生效。

选择设备上的电话，进入：

电话类型选择相应的话机类型.然后下一页。

选择SCCP协议，注：Cisco SCCP：瘦小客户端控制协议。瘦小客户端控制协议（SCCP：Cisco Skinny Client Control Protocol）是用于思科呼叫管理器及其 VOIP 电话之间的思科专有协议。有些供应商也支持该协议。

第三方SIP话机

第三方SIP话机注册方法与手动注册基本相同，主要区别在于选择话机类型的时候需要选择Third-party SIP Device

Advanced是给视频使用

其余使用Basic即可

然后在End User下创建一个用户用户SIP的验证。

在用户下点击Device Association后，选择之前创建的SIP 话机，保存即可。

之后在SIP话机上填写CUCM地址已经验证的用户名密码，用户名密码即End User下绑定了该SIP话机的用户名密码。

三、添加网关

进入到CM Administration下 进入到Gateway选项中

点击Add New，选择H.323 Gateway。其余那些有型号的都是MGCP协议。

H.323最简单，最易用，非常便于排错，是工程的上选。而且H.323支持TCL脚本，可以方便的实现Auto Attendant。

MGCP配置复杂，不好用，排错困难，但支持AAR。MGCP不支持TCL脚本。

另外我们还可以选择SIP或者SCCP。SIP要单独收费，占用license，没有必要。SCCP还是类似MGCP的结构，不好用。

之后填写网关的IP地址保存即可。

四、添加用户

手动添加

进入到CM Administration下 进入到End User选项中

Add New，填写基本信息，保存即可。

LDAP同步

进入到CM Administration下 进入到LDAP选项中

首先进入LDAP System，激活LDAP同步。LDAP同步被激活后，手动添加的用户将无效。

之后进入LDAP Directory

填写一些用户的信息

LDAP Server information下填写LDAP的IP地址

五、Extension Mobility配置

激活Cisco Extension Mobility

在CUCM右上角的下拉菜单选择Cisco Unified Serviceability，然后依次选择

Tools—Service Activation ,激活Cisco Extension Mobility

如需取消Cisco Extension Mobility,则需要取消激活该服务。

创建Cisco Extension Mobility 服务

在CUCM ADMIN 页面，依次选择Device->Device Setting ->Phone Services,然后输入服务名称：LOG IN/LOG OUT（任选），在URL栏输入http://:8080/emapp/

EMAppServlet?device=#DEVICENAME#，即在本配置中为注意该URL为大小写敏感

类型为XML ，最后选择Save。

设置服务参数

在CUCM ADMIN页面，依次选择Systemà Service Parameters。设置参数如下

Enforce Maximum Login Time 强制最大登陆时间

选择True ，指定登陆时间，如需进行自动退出，必须配置该参数。

Multiple Login Behavior 多点同时登陆行为

可以指定Multiple Logins Allowed（允许同时多点登陆）

Multiple Logins Not Allowed（禁止同时登陆同一用户）

Auto Logout(当用户在其他电话上登陆时，当前电话选择自动退出)

创建Device Profiles

在CUCM ADMIN 选择页面，依次选择Device->Device Setting->Device Profiles->Add New

新建一个Device Profile ，输入Device Type ,协议类型，Device Pro 以及电话按键模板，选择保存。最后为该Device Profile 配置一个DN ，共用户加载该Device Profile 时，获得DN号码。

创建终端用户帐号，并关联Device Profile

在CUCM ADMIN 页面，依次选择User ManagermentàEnd User->Add New

输入 USER ID 、PASSWORD PIN码，并为添加Device Profile至 Controlled Profiles进行关联。注意不要将该用户账号关联至物理设备，否则将在物理电话存在时，因为服务参数的Multiple Login Behavior 为not Allowed 导致用户无法登陆。

为电话绑定服务

在Devie–>Phone àfind ,找到注册到CUCM的电话，选择该电话后，勾选Enable Extension Mobility ，并选择 use current Configuation。在Log out profile 处，选择当用户退出后，使用的默认的Device Profiles。最后，为了绑定服务至电话，选择CUCM右上角的下拉菜单，选择Subscribe/Unsubscribe Services，为该电话绑定2.2处新建的LOGIN/LOG OUT电话服务。

为了让用户可以在登陆后，使用退出service，此时，在关联的Device Profile 处也应绑定Logg in Log out服务。

参考Cisco官方文档

七、MRGL配置

点击Media Resources—Media Resource Group

起名叫MRG，把所有Available的都选择上。保存。
这里都是系统默认的资源，简单解释一下：
ANN就是Annunciator。提供的资源是你拨错号码告诉你您拨打的号码有错误等。
CFB就是Conference Bridge。会议桥是作多方会议必备的。如果不能做Ad Hoc或者Meet-Me会议，查看有没有给话机分配会议桥资源。
MOH就是Music On Hold。就是你把人Hold了，对方能听到音乐，而不是无声。
MTP就是Media Termination Point。你做transfer，call park什么的，都需要MTP来暂时保管RTP数据流。

点击Media Resources—Media Resource Group List

起名字叫MRGL，把刚才配好的MRG调用，保存。

九、Hunt Group

进入到CM Administration页面下，进入到Route/Hunt下

首先建立一个Line Group 这里可以选择这个组里分配了哪些分机号。已经振铃规则（自上向下、循环、最长空闲时间、共振）等

十、Meet Me

进入到CM Administration下的Meet-Me Number/Pattern中


点击Add New，填写一个MeetMe的号码。也可以通过例如666X来创建6660-6669十个MeetMe的号码。

十一、CSS Partition 呼叫控制

CSS 全称是 Calling Search Space。可以理解成一个钥匙串。而Partition可以理解成一个锁。一个CSS下可以拥有一个、多个或者没有Partition。

每个号码可以给他一个Partition（锁）或者为None，None即没锁

每个设备或者号码可以拥有一个CSS（钥匙串），来控制该设备或者号码是否拥有拨打另一个号码的权限。如果该设备或者号码下的CSS拥有被叫号码的partition的，那么电话可以被打通，否则不能。

这里举个例子：

分别有1000-3999那么多分机号，他们的Partition和CSS分别如下：

DN Partition CSS
1000-1999 None None
2000-2999 Local CSS-Local
3000-3999 International CSS-International

CSS-Local和CSSS-international包含的Partition分别如下：

CSS Partition
CSS-Local Local
CSS-International Local和International

那么在这种情况下：

由于1000-1999的CSS是None，因此无法拨打任何一个有Partition的分机，所以这个分机号段的电话只能拨打1000-1999这个段的分机。

2000-2999这个分机段的CSS是CSS-Local，这个CSS拥有Local这个Partition，而没有International，所以无法拨打3000-3999。因此这个分几段的分机就只能拨打1000-2999的分机。

3000-3999这个段的分机的CSS是CSS-international，这个CSS拥有Local和International两个Partition，因此他可以拨打1000-3999所有的分机。

十二、相关小细节

1、CDR

开启是CDR功能后，CUCM将记录所有CUCM产生的话务信息。一般如果需要话务报表，则需要打开此功能。

2、G.722

如果CUCM中有使用BIB的录音，那么需要将G722关闭，BIB录音方式不会对G.722进行录音，可以进入到Service Parameter下进入Cisco Callmanager

将G722 Codec Enable改成Disable或者Enable for All devices except recording-enable devices

3、Max conference

CUCM默认的最大conference为4方，可以对CUCM的参数修改来修改最大方数。

进入到Service Parameter下进入Cisco Callmanager

修改Ad Hoc修改conference的最大方数 MeetMe 修改MeetMe的最大方数

4、CUCM Hostname/IP address

如果环境中没有对CUCM做解析的DNS，那么一般情况下可以将CUCM的Hostname直接改成IP地址，修改方法，进入到CM Administration的System→Server下，将Hostname改成IP地址即可。

5、Date/Time

如果发现系统时间正常，而话机时间不正常的情况，一般是差8个小时，这种情况一般是由于没有设置时区导致的。可以进入到Data/time Group中进行时区的设置。

6、License Upload

在CM Administration页面下，点击License ，将license文件导入即可。导入后可以通过License Unit Report来查看当前的License使用情况。

7、NTP Server

首先进入到OS Administration页面下，进入NTP Servers

点击Add New添加新的NTP服务器。

之后可以查看到各个NTP Server的状态

8、Softkey


设备-设备设置-软键模板

进入：

点击查找

此处可以修改或者新建一个新的Soft key 的模版

右上方选择配置软键布局，点击转至对模版进行编辑。

在这边可以添加话机各个状态下的软件按钮。

然后把所有需要使用该模版的话机下的软件模版选项选成该软件模版，之后重置话机。

常见术语

CUCM：Cisco Unified Communications Manager
CUBE：Cisco Unified Border Element
FXS（Foreign Exchange Station）模拟用户接口
可以理解为计算机通信中的DCE（数据通信设备）接口。用来连接具有FXO端口的端设备.FXS接口用于直接连接电话机;连接PBX的外线，用FXS接口向PBX中继外线提供馈电信号
FXS—外围交换用户话机接口是将模拟线路传输到话机的埠。换句话说，就是传送拨号音，电池电流以及响铃电压。
FXO（Foreign Exchange Office）模拟中继接口
可以理解为计算机通信中的DTE（数据终端设备）接口。连接具有FXS端口的电话交换机设备，FXO接口用于连接PSTN，用于连接PBX分机内线，由PBX向FXO接口提供馈电信号。
FXO—外围交换局接口是接受模拟线路的埠。它是电话或传真机，或模拟电话系统上的插口，用来传输挂机/摘机指示(回线闭合)。由于FXO端口附着于装置上，如传真机或电话机，所以这种装置通常被称为“FXO装置”。
FXO和FXS总是相辅相成的，类似插头和插座的关系。
